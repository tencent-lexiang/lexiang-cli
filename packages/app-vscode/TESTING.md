# 手动测试计划：乐享知识库拖拽功能

## 1. 测试环境

- VS Code 1.85+
- 已安装并启用 LeFS 插件
- 已登录并挂载至少一个乐享知识库

## 2. 测试场景

### 场景 A：拖拽文件到编辑器

**前置条件**：打开乐享侧边栏，展开知识库目录。
**步骤**：

1. 鼠标左键按住侧边栏中的某个文档（`.lxdoc` 或其他文件）。
2. 拖拽到 VS Code 右侧的编辑器区域。
3. 释放鼠标。
**预期结果**：

- 编辑器应打开该文档的内容（只读模式或预览模式）。

### 场景 B：拖拽文件到终端

**前置条件**：打开 VS Code 终端 (`Ctrl+` `)。
**步骤**：

1. 鼠标左键按住侧边栏中的某个文档。
2. 拖拽到终端窗口区域。
3. 释放鼠标。
**预期结果**：

- 终端输入框中应自动填入该文档的路径（`lxdoc://...`）。

### 场景 C：拖拽文件到 Chat 输入框

**前置条件**：打开 Copilot Chat 或 CodeBuddy Chat 对话框。
**步骤**：

1. 鼠标左键按住侧边栏中的某个文档。
2. 拖拽到 Chat 输入框中。
3. 释放鼠标。
**预期结果**：

- Chat 输入框应识别该文件为上下文引用（通常显示为文件芯片或路径）。
- 发送消息后，AI 应能读取该文件内容。

### 场景 D：多选拖拽

**步骤**：

1. 按住 `Cmd` (Mac) 或 `Ctrl` (Windows) 键，点击选择多个文档。
2. 拖拽选中的多个文档到编辑器或终端。
**预期结果**：

- 如果拖拽到编辑器，应分别打开多个编辑器标签页（视 VS Code 版本行为而定）。
- 如果拖拽到终端，应粘贴多个路径，以换行符分隔。

## 3. 异常测试

- 尝试拖拽文件夹：预期可能仅支持路径粘贴，或无响应（视实现而定，当前实现仅支持含 resourceUri 的节点）。
- 尝试拖拽到不支持 Drop 的区域：鼠标指针应显示禁止图标。
